# Simulation Results

This directory contains the raw simulation data exported directly from the Xilinx Vivado RTL testbenches. These CSV files capture the cycle-accurate behavior of the Systolic Array and its AXI wrapper during matrix multiplication (MAC) operations.

These logs serve as empirical proof of the mathematical correctness, Q-format saturation logic, and AXI-Stream synchronization of the design.

## File Manifest

The simulation logs are divided into two categories: **Bare-Metal Core** (bypassing AXI wrappers) and **System-Level** (via AXI-Lite and AXI-Stream).

### 1. Bare-Metal Core Simulation
Generated by `tb_systolic_core_engine.v`. These files log the raw data entering and exiting the core processing elements.
* **`core_input_stimulus.csv`**: Contains the 1D input vectors (Activations) injected into the core per clock cycle.
* **`core_mac_results.csv`**: Contains the output vectors (Results) after the 15-cycle wavefront latency, including the propagated `TLAST` signal.
  * *Test Scenario:* Multiplies input vectors with an Identity Matrix and adds a constant bias of `15`.

### 2. System-Level (AXI Wrapper) Simulation
Generated by `tb_systolic_axi_wrapper.v`. These files log the data transiting through the AXI-Stream interfaces, simulating a DMA transfer.
* **`sim_input_stimulus.csv`**: Contains the 128-bit AXI-Stream packets sent to `s_axis_tdata`.
* **`sim_mac_results.csv`**: Contains the 128-bit AXI-Stream packets received from `m_axis_tdata`.
  * *Test Scenario:* Multiplies input vectors with an Identity Matrix and adds a constant bias of `10`.

## How to Interpret the Data

### Columns
* **`Time(ns)`**: The exact simulation time the data was sampled (triggered on the rising edge of the clock when `VALID` and `READY` are both asserted).
* **`A0-A7` / `CH0-CH7`**: The 8 parallel input elements (16-bit each).
* **`Y0-Y7`**: The 8 parallel output elements (16-bit each).
* **`TLAST`**: The AXI-Stream boundary marker. A value of `1` indicates the end of a vector packet.

### Data Format (Q6.10 Fixed-Point)
All numerical values in these CSVs are represented as integer equivalents of the **Q6.10 fixed-point** format. 
To convert these raw integers back to floating-point values for software comparison, divide the values by `1024` ($2^{10}$).

For example, an input value of `1024` represents exactly `1.0`.

## Usage

These CSV files are designed to be easily parsed by data analysis tools. You can use Python libraries such as **Pandas** and **Matplotlib/Seaborn** to plot the hardware latency, verify the throughput, or compare the fixed-point RTL results against a floating-point PyTorch/NumPy software model.
